# Closed-Loop Firstmate Lifecycle Implementation Plan

> **For agentic workers:** REQUIRED: Use the existing Firstmate isolation and
> validation workflow. Steps use checkbox syntax for tracking.

**Goal:** Make Firstmate autonomously drive every task from admission through
> liveness supervision and terminal closeout without orphaned active work.

**Architecture:** Add one backend-neutral lifecycle library and command surface
> backed by append-only event receipts plus atomic per-task current-state files.
> Existing `.meta`, `.status`, backlog, watcher, and Treehouse surfaces become
> projections/adapters of that lifecycle contract. Reaping is idempotent and
> fail-closed: it can classify anything automatically, but destructive worktree
> removal requires clean/unleased proof and an explicit apply mode.

**Tech Stack:** POSIX Bash, jq, git, Treehouse CLI, existing Firstmate shell
> test harness.
